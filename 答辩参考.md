# 🎓 TasteWise — 开源软件技术课程答辩参考

> 本文档为 TasteWise 项目答辩提供素材支撑，涵盖项目定位、亮点提炼、对比分析、技术深度、开源协作展示等维度。可根据答辩时间灵活选用。

## 一、一句话定位

> **TasteWise 是一个专为本校食堂设计的菜品推荐系统，解决学生"今天吃什么"的选择困难。**

---

## 二、项目背景与痛点

### 2.1 痛点分析

| 痛点 | 说明 |
|------|------|
| 😖 **选择困难** | 食堂窗口多、菜品杂，每天纠结吃什么 |
| 📱 **外卖软件帮不上** | 美团/饿了么只收录商业餐厅，**没有基本伙食堂数据** |
| 🧠 **缺乏个性化** | 食堂菜谱固定，没有人根据你的口味做推荐 |
| 🔄 **反馈无门** | 觉得好吃/不好吃，没有地方记录和影响后续推荐 |

### 2.2 我们的方案

TasteWise 用 **"五味画像"（酸、甜、苦、辣、咸）** 为每一道菜品建模，用户通过滑块输入自己的口味偏好，系统匹配出最合适的菜品并解释理由。

## 四、项目亮点（推荐选 3-4 个重点讲）

### 亮点 1：五味文化 + 可解释推荐 🧬

- 用中国饮食文化中的"五味"（酸、甜、苦、辣、咸）作为推荐维度
- 比起"五星评分"更符合食堂场景的认知
- 每条推荐附带**自然语言理由**（如："辣味与你的高偏好接近"）
- 用户知道"为什么推荐这个"，增强信任感

### 亮点 2：地域口味融合 🗺️

- 同学来自五湖四海，口味习惯不同
- "老家是哪里"功能自动融合地域饮食画像
- 川渝偏好辣、江浙偏好甜、广东偏好清淡……
- 让异乡学子也能吃到熟悉的味道

### 亮点 3：双算法引擎可选 ⚡

| 算法 | 特点 | 适用场景 |
|------|------|---------|
| **加权欧氏距离** | 精确匹配，差异敏感 | "我就要辣度4的菜" |
| **余弦相似度** | 趋势匹配，忽略幅度 | "我喜欢偏辣的菜就行" |

- 用户可在侧边栏一键切换
- 加权欧氏距离支持五味重要性权重调整

### 亮点 4：纯 Python + 低门槛 🐍

- **技术栈**：Python + Streamlit + pandas + numpy
- 环境搭建只需 `pip install -r requirements.txt`
- 数据用 CSV 存储，Excel 可直接编辑
- **适合高校教学和二次开发**

### 亮点 5：数据抽象层，为后续扩展预留 🔌

```python
class CsvDataStore:    # 当前实现
    def load_dishes(self): ...
    def append_interaction(self): ...

class SqlDataStore:    # 未来实现（接口一致）
    def load_dishes(self): ...
    def append_interaction(self): ...
```

- 数据访问接口统一，切换数据库**不改业务代码**
- 体现软件工程中的"依赖倒置"原则

### 亮点 6：开源协作模式 🧑‍🤝‍🧑

- 团队 5 人，各有分工（文档、算法、测试、数据、UI）
- 使用 GitHub Issues + PR + Code Review 标准流程
- CI 自动运行测试和 Lint 检查
- MIT 许可证，任何人都可自由使用和贡献

## 七、常见答辩问题与回答

### Q1：和美团有什么区别？美团不能搜食堂吗？

> 美团等外卖平台的核心模式是 **"商家入驻 → 用户下单 → 骑手配送"**，它们收录的是商业餐厅，不包含高校基本伙食堂。即使有些学校有外卖柜，美团上也只能看到商业档口的餐品，看不到基本伙窗口（如"基本伙周菜谱窗口"的麻婆豆腐、西红柿鸡蛋这类日常菜品）。TasteWise 专门针对本校食堂场景设计，覆盖基本伙和风味窗口的全部菜品。

### Q2：这个项目是课程作业还是真实项目？

> 两者都有。它始于开源软件技术课程，但定位是**可以真实使用的工具**。开发团队都是本校学生，每天都在食堂吃饭，自己也在用这个软件做决策。遇到不好用的地方会立即修改——这是纯粹的课设项目做不到的。

### Q3：为什么用 CSV 不用数据库？

> MVP 阶段优先追求 **快速迭代和低门槛**。CSV 的好处是：① 零配置，git clone 就能跑；② 用 Excel 就能编辑数据；③ 容易做数据备份和版本管理。同时代码层已经抽象了数据访问接口（`CsvDataStore`），后续切换到 SQLite/MySQL **不需要改业务代码**。

### Q4：推荐算法的准确度如何？

> TasteWise 采用的是 **基于内容的推荐（Content-Based Filtering）**，匹配度反映的是"菜品五味画像与用户偏好的接近程度"，这是一个**可解释的相似度分数**，不是传统意义上的"预测准确率"。我们的测试覆盖了精确匹配、Top-K 返回、价格边界等场景，确保算法在各种输入下行为正确。后续可以通过用户反馈数据训练模型，进一步提升推荐效果。

### Q5：项目的开源体现在哪里？

> ① **MIT 许可证**，任何人都可以自由使用、修改、分发；② **GitHub 公开仓库**，完整的开发历史可见；③ **贡献指南**（CONTRIBUTING.md），欢迎外部贡献；④ **Code Review + CI** 保证代码质量；⑤ 项目的 PRD、TECH_SPEC、算法文档全部公开。

### Q6：你们和商业推荐系统（抖音、淘宝）的区别？

> 商业推荐系统是"黑盒"的——用户不知道为什么看到某个内容。TasteWise 强调**可解释性**：每一条推荐都附带理由，用户知道为什么这道菜匹配自己的口味。这在食堂场景中特别重要——学生点菜是一个**有意识的决策行为**，不是无意识的刷信息流。

```

3. 快速设置五味偏好（酸3、甜2、苦1、辣5、咸4）（20秒）

5. 点击"开始智能推荐" → 展示推荐结果（15秒）
```
