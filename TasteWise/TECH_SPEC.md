## 1. 项目概述

TasteWise 是一个基于**五味偏好（酸、甜、苦、辣、咸）** 的高校食堂菜品推荐系统。用户通过评分方式表达对每种口味的偏好程度，系统基于评分匹配算法推荐最符合用户口味的菜品。

### 3.4 推荐算法

完整算法说明见 [`ALGORITHM.md`](ALGORITHM.md)，包括输入输出、筛选流程、Top-K 排序、异常处理和测试覆盖。

#### 算法 A: 加权欧氏距离

```python
score = (1 - distance / max_distance) * 100
# 其中 distance = sqrt(sum(weights * (user - dish)^2))
# max_distance = sqrt(sum(weights * 4^2))
```

- **特点**: 强调口味数值的绝对接近程度
- **输出**: 0~100 的匹配度百分比
- **支持权重**: 每种口味可设置不同重要性

#### 算法 B: 余弦相似度

```python
similarity = dot(user, dish) / (norm(user) * norm(dish))
score = similarity * 100
```

- **特点**: 强调口味偏好方向的相似性（趋势一致即高分）
- **输出**: 0~100 的匹配度百分比
- **不支持权重**: 余弦相似度天生归一化

## 5. UI 交互流程

```
用户打开页面
    ↓
设置五味偏好 (1~5滑块)
    ↓
(可选) 设置权重 / 筛选食堂 / 价格范围 / 选算法
    ↓
点击"开始智能推荐"
    ↓
recommender.recommend_dishes()
    ├─ 校验输入
    ├─ 过滤 (食堂 / 价格)
    ├─ 遍历菜品计算匹配度
    │   ├─ 加权欧氏距离 OR 余弦相似度
    │   └─ 生成推荐理由
    └─ 排序取 Top-K
    ↓
展示推荐结果卡片
    ├─ 菜品名 + 食堂 + 窗口 + 价格
    ├─ 匹配度进度条 + 百分比
    ├─ 推荐理由
    └─ 交互按钮 (喜欢 / 不喜欢 / 收藏)
```

## 9. 技术决策记录

### 9.1 为什么选 CSV 而不是 SQLite/MySQL？
- **MVP 阶段**：数据量小（60 条菜品），CSV 无需配置数据库环境
- **可读性**：直接用 Excel 或文本编辑器可编辑
- **迁移路径**：后续可平滑迁移到 SQLite，数据结构基本不变

### 9.2 为什么选 Streamlit 而不是 Flask/Django？
- **开发效率**：Streamlit 用纯 Python 写 UI，无需手写 HTML/CSS/JS
- **交互密度**：推荐系统的滑块、筛选器、按钮等组件 Streamlit 原生支持
- **部署简单**：单文件即可运行

### 9.3 为什么固定五味而不是 EAV？
- **领域约束**：菜品口味的核心维度就是"五味"，短期内不会变化
- **性能**：EAV 模式需要 JOIN 查询，在 60 条数据量下得不偿失
- **代码简洁**：固定列名使算法实现更直观
