## 功能简介

| 功能 | 说明 |
|------|------|
| 🎯 **五味偏好设置** | 通过滑块为酸、甜、苦、辣、咸打分（1~5） |
| ⚖️ **五味权重调整** | 设置每种口味在推荐时的重要性 |
| 🏫 **食堂筛选** | 按食堂筛选菜品（一食堂/二食堂/三食堂/风味食堂/智慧食堂） |
| 💰 **价格筛选** | 设置价格范围过滤菜品 |
| 🔀 **双算法支持** | 可选「加权欧氏距离」或「余弦相似度」 |
| 📊 **匹配度展示** | 每条推荐显示 0~100% 的综合匹配度 |
| 💡 **推荐理由** | 每条推荐附带可解释的理由 |
| 👍 **交互反馈** | 支持「喜欢」「不喜欢」「收藏」反馈记录 |
| 📝 **饮食记录** | 用户交互行为自动记录到 CSV 文件 |

## 快速开始

### 一键启动（推荐）

**Windows (cmd)** — 直接双击 `start.bat`，或执行：
```batch
start.bat
```

**Windows (PowerShell):**
```powershell
.\start.ps1
```
> 如果提示"无法加载文件"，先执行：`Set-ExecutionPolicy -Scope Process Bypass`

**macOS / Linux:**
```bash
chmod +x start.sh
./start.sh
```

启动脚本会自动完成：✅ 检测 Python → ✅ 创建虚拟环境 → ✅ 安装依赖 → ✅ 启动应用。

启动成功后，终端会输出：
```
Local URL: http://localhost:8501
```

**浏览器会自动打开这个地址**，那就是 TasteWise 的界面。

> 💡 **http://localhost:8501 是什么意思？**
> - `localhost` = 你的**本机**（127.0.0.1），数据只在你自己电脑上传输，不会传到外网
> - `8501` = **端口号**，类似"第8501号通道"
> - 合起来就是：在你**自己的电脑**上，通过 **8501 号端口**访问应用
> - 如果浏览器没有自动弹出，手动输入 `http://localhost:8501` 即可

### 手动分步启动

如果希望手动控制每一步：

```bash
# 1. 进入项目目录
cd TasteWise

# 2. 创建虚拟环境
python -m venv .venv

# 3. 激活虚拟环境
#    Windows (cmd):  .venv\Scripts\activate.bat
#    Windows (PS):   .\.venv\Scripts\Activate.ps1
#    Mac/Linux:      source .venv/bin/activate

# 4. 安装依赖
pip install -r requirements.txt

# 5. 启动应用
streamlit run app.py
```

### 停止应用

在终端中按 `Ctrl + C`，或直接关闭启动脚本的窗口。

## 数据管理

### 菜品数据（dishes.csv）

系统内置了 **878 道菜品**，覆盖 2 个食堂（一食堂、二食堂）。位于 `data/dishes.csv`。

**添加新菜品：**
直接用 Excel 或文本编辑器打开 `data/dishes.csv`，新增一行：

```csv
dish_id,name,canteen,window,price,acid,sweet,bitter,spicy,salty
61,新菜品名,一食堂,川菜窗口,10,3,3,1,4,3
```

> 注意：`dish_id` 不能重复，`price` 和五味评分只接受数字。

**修改菜品：**
找到对应行，直接修改字段值后保存即可。

**删除菜品：**
删除对应行，重启应用后生效。

### 用户交互记录（interactions.csv）

用户点击「喜欢」「不喜欢」「收藏」时自动写入 `data/interactions.csv`，无需手动编辑。

### 用户数据（users.csv）

预置了一位演示用户，可手动添加更多用户：

```csv
user_id,username,acid,sweet,bitter,spicy,salty
zhangsan,张三,5,3,1,4,2
```

## 推荐算法

系统支持两种推荐算法，可在侧边栏切换。

如果选择了"老家是哪里"，系统会在用户手动设置的五味偏好基础上，轻微融合常见地域口味画像，并对菜名、窗口、食堂中包含地域菜品线索的结果给予小幅加分。这样既保留用户当前偏好，也能让推荐更贴近家乡饮食习惯。

### 加权欧氏距离（默认）

强调口味数值的**绝对接近程度**。用户和菜品的五味画像分别为向量 u 和 d：

```
distance = sqrt( Σ weight_i × (u_i − d_i)² )
match_score = (1 − distance / max_distance) × 100
```

- ✅ 适合：希望口味**数值完全匹配**的场景
- ⚡ 支持：五味重要性权重调整

### 余弦相似度

强调口味偏好**方向的一致性**（如"喜欢辣的菜"比"辣度恰好是 4"更重要）：

```
similarity = (u · d) / (‖u‖ × ‖d‖)
match_score = similarity × 100
```

- ✅ 适合：关注口味**整体趋势**而非精确数值
- ⚠️ 不支持：权重调整（余弦相似度天然归一化）

## 常见问题排查

### Q: 启动时报 `FileNotFoundError: 找不到菜品数据`

```
确保在 TasteWise 目录下运行：
  cd TasteWise
  streamlit run app.py

检查 data/dishes.csv 文件是否存在。
```

### Q: 浏览器打开后页面空白或报错

```
1. 检查终端是否有错误日志
2. 尝试刷新页面 (F5)
3. 确认端口 8501 未被占用：
   Windows: netstat -ano | findstr :8501
   macOS/Linux: lsof -i :8501
```

### Q: `pytest` 命令找不到

```
确保已激活虚拟环境并安装了依赖：
  pip install -r requirements.txt

或者使用 python -m pytest 代替 pytest。
```

### Q: 添加菜品后看不到新数据

```
重启 Streamlit 应用（Ctrl+C 停止，重新运行 streamlit run app.py）。
Streamlit 在启动时一次性加载数据。
```

### Q: git push 时报 SSL 连接错误

```
尝试使用以下命令（Windows Git Bash）：
  git -c http.sslBackend=openssl push origin <branch>

或切换为 SSH 方式：
  git remote set-url origin git@github.com:用户名/仓库名.git
```
